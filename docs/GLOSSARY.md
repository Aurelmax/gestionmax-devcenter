# Glossaire GestionMax DevCenter

**Tunnel SSH** : connexion sécurisée permettant d'accéder à une base distante en local.

**Process zombie** : processus orphelin qui continue de tourner en fond.

**Tauri** : framework permettant de créer des applications desktop avec un backend Rust et un frontend web.

**Invoke** : mécanisme Tauri pour appeler des fonctions Rust depuis React.

**Payload CMS** : backend Node que tu utilises dans tes projets (mais pas utilisé ici dans DevCenter).

**Next.js** : framework frontend (ton frontend principal).

**TailwindCSS** : système de classes utilitaires pour styler l'UI rapidement.

**React** : bibliothèque pour construire l'UI.

**Shadcn UI** : collection de composants React réutilisables basés sur Radix UI et Tailwind CSS.

**IPC (Inter-Process Communication)** : mécanisme de communication entre le processus frontend (React) et backend (Rust) dans Tauri.

**Hot reload** : rechargement automatique de l'application lors des modifications de code.

**Polling** : technique consistant à vérifier périodiquement l'état d'un système (toutes les 2 secondes dans notre cas).

**Toast** : notification temporaire affichée à l'écran pour informer l'utilisateur d'une action.

**Badge** : petit élément visuel affichant un statut ou une information (ex: "RUNNING", "STOPPED").

**Card** : composant d'interface représentant une carte contenant des informations.

**Dialog** : fenêtre modale permettant d'afficher des informations ou demander une confirmation.

**Hook (React)** : fonction permettant d'utiliser des fonctionnalités React (état, effets, etc.).

**TypeScript** : surcouche de JavaScript ajoutant le typage statique.

**Vite** : outil de build et serveur de développement rapide pour les applications web.

**Cargo** : gestionnaire de paquets et système de build pour Rust.

**Rust** : langage de programmation système rapide et sûr, utilisé pour le backend Tauri.

**Port** : numéro de port réseau sur lequel un service écoute (ex: 3000, 3010).

**PID (Process ID)** : identifiant unique d'un processus dans le système d'exploitation.

**JSON** : format de données textuelles utilisé pour la configuration des projets.

**ES Modules** : système de modules JavaScript moderne utilisant `import`/`export`.

**CommonJS** : ancien système de modules JavaScript utilisant `require`/`module.exports`.

**PostCSS** : outil de transformation CSS, utilisé avec Tailwind CSS.

**Dark mode** : thème sombre de l'interface utilisateur.

**Grid** : système de mise en page utilisant une grille pour organiser les éléments.

**Responsive** : capacité d'une interface à s'adapter à différentes tailles d'écran.

**Sandbox** : environnement isolé où le code s'exécute avec des permissions limitées.

**DevTools** : outils de développement intégrés au navigateur pour déboguer.

**Build** : processus de compilation et d'optimisation du code pour la production.

**AppImage** : format d'application portable pour Linux.

**Debian package (.deb)** : format de paquet pour les distributions Linux basées sur Debian.

**Executable (.exe)** : fichier exécutable pour Windows.

**Bundle identifier** : identifiant unique d'une application (utilisé pour les logs, etc.).

**Stdout** : sortie standard d'un programme (affichage dans le terminal).

**Stderr** : sortie d'erreur standard d'un programme.

**Linter** : outil analysant le code pour détecter les erreurs et problèmes de style.

**Type checking** : vérification des types TypeScript avant l'exécution.

**Hot Module Replacement (HMR)** : remplacement à chaud des modules modifiés sans recharger toute la page.

**State management** : gestion de l'état de l'application (données, UI, etc.).

**Component** : élément réutilisable de l'interface utilisateur en React.

**Props** : propriétés passées à un composant React.

**Event handler** : fonction appelée lors d'un événement (clic, changement, etc.).

**Async/await** : syntaxe JavaScript pour gérer les opérations asynchrones.

**Promise** : objet JavaScript représentant une valeur qui sera disponible dans le futur.

**Error handling** : gestion des erreurs dans le code.

**Logging** : enregistrement d'informations pour le débogage et le suivi.

**Configuration file** : fichier contenant les paramètres de configuration de l'application.

**Environment variable** : variable d'environnement système utilisée pour la configuration.

**Dependency** : bibliothèque ou package externe utilisé par le projet.

**Dev dependency** : dépendance utilisée uniquement en développement.

**Production build** : version optimisée de l'application pour la distribution.

**Development mode** : mode de développement avec outils de débogage et hot reload.
